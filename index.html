Basic git's

  • git config --global user.name "Kate" - вказує ваше ім'я
  • git config --global user.email sadamochka@gmail.com - вказує вашу електронну адресу
  • git init - створює новий репозиторій
  • git status - відображає стан робочого каталогу та розділу проіндексованих файлів
  • git add - додає зміну з робочого каталогу до розділу проіндексованих файлів
  • git commit - знімок поточного стану змін, доданих до розділу проіндексованих файлів
  • git log - перераховує комміти, зроблені в репозиторії в зворотному до хронологічного порядку
  • git checkout - дозволяє переміщатися між гілками, створеними командою git branch
  • git branch - дозволяє створювати, переглядати, перейменовувати та видаляти гілки
  • git push - вивантаження вмісту локального репозиторію у віддалений репозиторій
  • git pull - вилучення та завантаження вмісту з віддаленого репозиторію та негайного оновлення локального репозиторію цим вмістом
  • git merge - злиття окремих напрямків розробки, створених за допомогою команди git branch, в єдину гілку
  • git clone - для вибору існуючого репозиторію та створення його клону, тобто копії

Файл із розширенням файлу GITIGNORE

Файл Git Ignore, який використовується з системою керування версією / джерелом під назвою Git. Він визначає, які файли та папки не слід ігнорувати у певному вихідному коді. Його можна використовувати на контексті, так що правила застосовуються лише до певних папок, але ви також можете створити глобальний файл GITIGNORE, який застосовується до кожного сховища Git.

Advanced

  • git revert - операція для безпечного скасування змін, дія якої спрямована у майбутнє
  • git reset - команда, що використовується для скасування локальних змін у репозиторії Git
  • git rebase - можете взяти всі комміти з однієї гілки і в тому ж порядку застосувати їх до іншої гілки
  • git cherry-pick - можна вибірково застосувати коміти Git до поточної робочої гілки HEAD
  • git stash - дозволяє на якийсь час відкласти зміни, зроблені в робочій копії, щоб ви могли застосувати їх пізніше
